ezAR Startup Kit Cordova Plugin

Develop augmented reality applications using the ezAR Startup Kit. The ezAR Startup Kit includes
4 ezAR Cordova plugins, example projects and getting started documentation. ezAR functionality
is divided among the following plugins:

com.ezartech.ezar.videooverlay - overlay camera view with Cordova WebView
com.ezartech.ezar.facedetector - detect faces in the videooverlay plugin's camera view
com.ezartech.ezar.flashlight - control the device light
com.ezartech.ezar.snapshot - capture an image of the app user interface

You can mix and match the plugins based on the functionality of your app. The VideoOverlay plugin
positions a camera view below the Cordova WebView component. ezAR provides access to the back and front
device cameras. When a camera is started its output is displayed on the ezAR camera view. By default
the camera view is not visible as it is masked off by the WebView's HTML content. Increase the transparency
for the area(s) of the app UI (html) where you wish to see the camera view.

Notes

  • The VideoOverlay and Flashlight plugins must be initialized before use.
  • When using the Flashlight and VideoOverlay plugins together in an Android app, setting the light on
    may not immediately turn the light on. Instead the light will only turn on when the camera on the same
    side of the device as the light is running. This is due to the tight dependency between the camera and
    light operations on Android devices where the light will not turn on if the camera is not running.
  • The VideoOverlay plugin supports the optiona webviews such as Android Crosswalk. But iOS WKWebview is not yet support.

Supported Platforms

  • iOS 7 and greater
  • Android 4.2 and greater, standard webview and Crosswalk webview

Getting Started

The simplest ezAR application involves adding the VideoOverlay plugin
to your Corodva project using the Cordova CLI

  cordova plugin add pathtoezar/com.ezartech.ezar.videooverlay

Next in your Cordova JavaScript deviceready handler include the following
JavaScript snippet to initialize the VideoOverlay plugin and activate the
camera on the back of the device, i.e., the camera away from the display.

 ezar.initializeVideoOverlay(
     function() {
         ezar.getBackCamera().start();
     },
     function(err) {
         alert('unable to init ezar: ' + err);
 });
